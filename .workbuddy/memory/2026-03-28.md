# 2026-03-28 工作记录

## 图片上传功能修复

### 问题
- 小程序添加宠物时头像上传后显示本地临时路径（http://tmp/...）
- 管理后台宠物列表头像显示为默认图标

### 根本原因
1. **存储桶未创建** - Supabase Storage 中 `petmind` 存储桶不存在
2. **字段名不一致** - 代码用 `photo_url`，数据库用 `avatar`
3. **上传方式不可靠** - 使用 base64 转换上传
4. **编译缓存问题** - 代码修改后未生效

### 修复内容

#### 1. 小程序端 (src/utils/supabase.ts)
- 使用 `uni.uploadFile` 替代 base64 转换上传
- 添加详细日志便于调试

#### 2. 字段统一
修改以下文件将 `photo_url` 改为 `avatar`：
- `src/pages/pet-add/pet-add.vue`
- `src/pages/index/index.vue`
- `src/pages/pet-detail/pet-detail.vue`
- `app/admin/pets/page.tsx`

#### 3. 用户 ID 统一 (src/pages/pet-add/pet-add.vue)
添加宠物时优先使用登录用户 ID：
```javascript
const currentUser = uni.getStorageSync('currentUser')
const userId = currentUser?.id || getDeviceId()
```
